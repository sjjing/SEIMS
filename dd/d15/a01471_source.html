21 # 1. MongoDB安装
22 ## i. 安装
23 + 运行`mongodb-win32-x86_64-2.6.1.msi`，实际上是解压程序，将解压后的所有文件放到一个**全英文且没有空格**的文件夹中，比如`E:\MongoDB`
24 + 在MongoDB文件夹内新建logs文件夹，用于存储日志文件，比如`E:\MongoDB\logs`，并在此文件夹内新建空文件`mongodb.log`
25 + 在MongoDB文件夹内新建db文件夹，用于存放数据库文件，比如`E:\MongoDB\db`
26 + 启动MongoDB:
27  以系统管理员身份运行cmd，输入以下命令：
28 
29  ```shell
30  cd E:\MongoDB\bin
31  E:
32  mongod.exe --dbpath=e:\mongodb\db
33  ```
34 
35 
36  如看到控制台最后一行类似:
37 
38  ```
39  Tue Oct 09 11:50:55 [websvr] admin web console watiing for connections on port 27017
40  ```
41 
42  说明启动成功,MongoDB占用系统27017端口。
43 
44 ## ii. 测试
45 测试MongoDB，以管理员身份新建一个cmd窗口：
46 
47 ```shell
48 cd E:\MongoDB\bin
49 E:
50 mongo
51 ```
52 
53 如出现`connecting to:test`说明测试通过。
54 
55 ![](http://i.imgur.com/Wkr40Z2.png)
56 
57 继续测试：
58 
59 ```shell
60 use test
61 db.foo.save({hello:0})
62 db.foo.find()
63 ```
64 
65 如果出现类似`{ "_id" : ObjectId("574cfef6c02688f96a5819a1"), "Hello" : 0 }`之类信息，说明测试成功数据已经插入数据库，
66 
67 ![](http://i.imgur.com/Zz9caSo.png)
68 
69 然后输入`exit`退出。

71 # 2. MongoDB配置
72 
73 + **注册MongoDB为系统服务**
74  以系统管理员身份运行cmd（必须以系统管理员身份运行cmd，否则会报错）：
75 
76  ```shell
77  cd E:\MongoDB\bin
78  E:
79  mongo.exe --dbpath=d:\MongoDB\db --logpath=d:\MongoDB\logs\mongodb.log --install --serviceName "MongoDB"
80  ```
81 
82  如果控制台出现类似 `Tue Oct 09 12:05:15 Service can be started from the command line with 'net start MongoDB'`这样的语句，说明服务已经注册成功。
83 
84 + **启动MongoDB数据库服务**
85  在cmd中输入:
86 
87  ```shell
88  net start MongoDB
89  ```
90  即可启动MongoDB数据库服务，此时控制台输出Mongo DB服务已经启动成功，说明系统启动成功。
91 
92  注意：MongoDB的安装目录需全英文且没有空格
93 
94 + **MongoDB 安装为Windows服务，设置为自动启动**
95 +
96  以管理员方式运行cmd,输入以下命令:
97 
98  ```shell
99  mongod --install --serviceName "MongoDB" --serviceDisplayName "MongoDB" --logpath d:\MongoDB\logs\mongodb.log --logappend --dbpath d:\MongoDB\db --directoryperdb
100  ```
101 
102  不提示错误即表示设置成功！
103 
104  `--install`：安装MongoDB服务
105 
106  `--serviceName`：安装Windows服务时使用的服务名
107 
108  `--serviceDisplayName`：在Windows服务管理器中显示的服务名
109 
110  `--logpath`：MongoDB日志输出文件名称。虽说该参数直译是“日志路径”，其实要指定的是一个具体的完整文件名。这里我使用的是C盘根目录下的MongoDB.Log文件。该文件不用事先创建，直接指定就是了。
111 
112  `--dbpath`：指定MongoDB数据存放的路径。这个就是最关键的参数了，不仅该目录要存在，并且最好不要以“\”结尾。
113 
114  `--directoryperdb`：这个参数很好理解，让MongoDB按照数据库的不同，针对每一个数据库都建立一个目录，所谓的“目录每数据库”。
